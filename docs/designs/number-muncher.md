## Problem Statement

A quick, simple, engaging browser game so the user's daughter can practice her
times tables. Inspired by MECC's Number Munchers (1986): grid of numbers, a
muncher character moved with arrow keys, space bar to munch numbers matching a
rule ("multiples of 7"). Fidelity to the original is explicitly NOT a goal;
being fun enough that she plays it voluntarily IS the goal.

## Research Facts (original game, verified via Wikipedia/MobyGames/Internet Archive)

- MECC, 1986, Apple II, designed by R. Philip Bouchard (The Oregon Trail).
- 5×6 grid. Muncher = arrow keys + space. NOT timed: lives-based (3 reserve),
  lose a life on wrong munch or Troggle contact. Extra life at score threshold.
- Level clears when ALL rule-matching numbers are munched. Troggle count
  escalates: 1 (levels 1–3), 2 (from level 4), 3 (from level 8).
- Troggle types: Reggie (straight lines), Bashful (random, flees Muncher),
  Smartie (chases Muncher).
- Safety squares: temporary squares Troggles cannot enter.
- Five modes: Multiples, Factors, Primes, Equalities, Inequalities.

## Accepted Scope (v1)

- Start screen: choose table(s) 2–12 (single or multi-select), choose mode
  (Classic / Blitz), name + muncher color settings, mastery stars per table.
- Grid: 6 columns × 5 rows (30 cells), all 30 cells filled every board.
- Multi-select rule: a number is correct if it is a multiple of ANY selected
  table (OR semantics). Wrong-munch explanations show the bracketing facts
  (nearest product below and above) from the single closest selected table —
  single-table: "44 isn't a multiple of 7 — 6×7=42, 7×7=49"; multi-select:
  "45 isn't a multiple of 7 or 8 — 5×8=40, 6×8=48". Stats in multi-select:
  munch results feed the game-over session-accuracy display ONLY — never the
  per-table rolling window (OV-7A); per-fact miss counts are recorded
  against the actual number munched.
- Board minutiae: duplicates may appear (a single table has only ~12 distinct
  products, so duplicates are expected); a wrongly-munched number STAYS on
  the board (it still must not be munched); a correctly-munched cell goes
  empty and stays empty (Classic).
- Classic mode: guaranteed ≥6 correct answers and ≥40% incorrect per board;
  near-miss distractors (products of adjacent tables, correct-product ±1);
  munch all correct numbers to clear the level. Scoring: 10 points per
  correct munch + 50 × level bonus on level clear; extra life every 1000
  points; 3 lives to start.
- Level progression: endless. Per level: Troggle tick interval decreases 5%
  (floor 350ms) and the product pool weights larger factors — at level n,
  factor k is drawn with weight 1 + max(0, k − (13 − n)) — always within
  factors 1–12 of the selected tables (never drifts past the facts being
  practiced).
- Game over (lives = 0, Classic): screen shows final score, level reached,
  and session accuracy, with Play Again (same settings) and Back to Start
  buttons.
- Wrong munch: lose a life AND show why ("44 isn't a multiple of 7 — 6×7=42,
  7×7=49"). Learning moment, not just punishment.
- Troggles (Classic mode only): one straight-line Troggle from level 1
  (slow), second from level 4. Spawn/respawn at a random edge cell ≥3 cells
  (Manhattan) from the Muncher, heading perpendicular-inward from its edge
  (random axis-aligned inward direction for corners). The Muncher starts
  each board at the center cell. A Troggle that walks off the board edge
  respawns per the spawn rule. Troggles move every tick (they can pass over
  numbers but never remove them, and cannot camp a cell — no deadlock with
  the clear condition). Contact = lose a life; cell-swap between ticks counts
  as contact. On contact the Muncher respawns IN PLACE with 1.5s
  invulnerability and the contacting Troggle respawns per the spawn rule.
- Blitz mode: 60 seconds, munch as many rule-matches as possible. No
  Troggles, no lives (decision F1/1A). Wrong munch = −5 seconds + the
  explanation, and the timer pauses while the explanation shows. 10 points
  per correct munch. When the last correct number is munched the board
  immediately reseeds and the timer keeps running. High scores are keyed by
  the sorted table-selection set (e.g. "7", "7+8"). Blitz munches update
  per-fact miss counts but NOT star accuracy (timer pressure must not
  sabotage mastery stars); blitz high scores are their own stat.
- Input: arrow keys + space on keyboard. Touch (decision OV-9A): tap ANY
  cell and the Muncher walks there step-by-step (Troggles remain dangerous
  en route); munching happens ONLY via a large on-screen MUNCH button — a
  mis-tap can never munch, so a fat finger never costs a life.
- Pause safety (decision OV-10A): 1.5s invulnerability on EVERY resume
  (explanation close, tab return, manual unpause); manual pause via P key /
  on-screen ⏸ button; explanations dismiss on explicit tap/keypress only,
  never auto-timeout.
- Audio: synthesized crunch / buzz / fanfare via Web Audio; mute toggle
  persisted.
- Progress: localStorage — per-table accuracy, per-fact miss counts, high
  scores, mastery stars per table, computed over a ROLLING window of the
  last 50 attempts per table (decision OV-6A — lifetime accuracy would
  punish early learning misses): 1★ = ≥20 attempts in window at ≥80%
  accuracy, 2★ = ≥35 attempts at ≥90%, 3★ = full 50-attempt window at ≥95%
  (thresholds are on ATTEMPT counts, not correct counts — "50 correct of the
  last 50" would silently demand perfection). Star accuracy accrues from single-table
  Classic sessions ONLY (decision OV-7A — multi-select debit attribution is
  guesswork; multi-select and blitz sessions still record per-fact miss
  counts and high scores). The start screen notes that stars are earned in
  single-table play.
- Celebrations: level-clear and new-high-score screens (confetti, big type).
- Repo: git init, initial commit, CLAUDE.md with gstack routing rules,
  README, TODOS.md, GitHub repo + Pages once `gh auth login` is redone.

## Key Implementation Decisions (temporal interrogation, resolved now)

1. Board seeding must GUARANTEE ≥6 correct answers and ≥40% incorrect; pure
   random seeding can produce degenerate boards (all-correct or one-correct).
2. Distractors must include near-misses: products from neighboring tables and
   correct-product ±1. Pure random distractors make the game trivially easy.
   2a. CRITICAL: the seeder must reject any candidate distractor where
   `isMatch(rule, n)` is true — near-miss generation can emit numbers that
   are actually correct (practicing 7s, adjacent-table product 6×7=42 IS a
   multiple of 7; the collision space grows with multi-select). Without this
   filter the game punishes correct math.
   2b. Degenerate-pool fallback: with many tables selected, the near-miss
   pool shrinks (most candidates match the rule). When fewer than the needed
   distractors survive the 2a filter, fill the remainder with random
   non-matching numbers in the same value range. The board must always
   satisfy decision 1's guarantees regardless of selection.
3. Game logic (`game.js`) is pure functions/state machine with zero DOM access
   so it is unit-testable and mode rules are pluggable (`isMatch(rule, n)`).
   Test runner: `node --test` against the pure modules (game.js, seeding,
   storage serialization). Module format: plain script tags in dependency
   order attaching to `globalThis`, with a guarded CommonJS export shim
   (`if (typeof module !== 'undefined') module.exports = ...`) on the pure
   modules — ES modules are blocked over `file://` in Chrome, and this keeps
   BOTH "double-click index.html" and `node --test` working with no build
   step. Fallback distribution before GitHub Pages is live (gh auth pending):
   `python3 -m http.server` on the LAN reaches the tablet.
4. Troggle movement runs on a tick timer (~700ms/step at level 1) independent
   of player input; grid re-render is diff-free full redraw (30 cells, cheap).
5. Wrong-munch feedback pauses Troggles while the explanation shows —
   otherwise the kid gets eaten while reading.
6. Touch targets ≥48px; layout must fit a 768px-wide tablet portrait.
7. localStorage failures (private browsing, quota, corrupted JSON on parse)
   degrade silently to in-memory session state / defaults — game never
   breaks because storage did.
8. Muncher name/color are runtime settings in localStorage, never hardcoded
   in the public repo. The name is rendered with `textContent` only, never
   `innerHTML` — closes the app's single injection vector (decision F4/4A).
9. AudioContext is created/resumed inside the first user-interaction handler
   (the start button) — browser autoplay policy otherwise silently blocks
   all sound (decision F2/2A).
10. input.js calls preventDefault() on arrows/space while game state is
    PLAYING (page must not scroll on munch) and ignores key auto-repeat —
    movement is one step per keypress (decision F3/3A).
11. Game pauses (Troggles frozen, blitz timer stopped) on the `visibilitychange`
    event when the tab is hidden — background-tab timer throttling would
    otherwise make blitz unfair and Troggles teleport on return.
12. Outside-voice decisions (cross-model review): OV-6A rolling star window,
    OV-7A single-table star accrual, OV-8B full v1 scope retained (playtest
    gate rejected — CC build speed makes build-complete-then-tune cheaper),
    OV-9A tap-to-walk + button-only munch, OV-10A pause safety bundle
    (invuln on every resume, manual pause, explicit explanation dismissal).
